# 依赖选型规则

## 什么时候应该优先引库

当需求属于通用复杂能力时，优先评估成熟开源方案：

- 表格
- 表单校验
- 拖拽
- 日期选择
- 图表
- 富文本
- 文件上传
- 认证

## 评估维度

1. 当前项目技术栈是否兼容
2. 是否仍在维护
3. 是否能明显减少自定义代码
4. 未来维护是否更轻
5. license 是否合适

## 不该引库的场景

1. 需求很小
2. 浏览器原生就够
3. 项目已有同类库
4. 只是为了“看起来更专业”

## 结论格式

做依赖建议时，尽量输出：

1. 当前技术栈
2. 当前需求类型
3. 推荐库
4. 安装命令
5. 推荐原因
6. 风险和 fallback
