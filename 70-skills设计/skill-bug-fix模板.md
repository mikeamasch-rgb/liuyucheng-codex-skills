# bug-fix

## 适用场景

- 页面报错
- 接口失败
- 行为与预期不一致
- 某个流程之前可用，现在失效

## 不适用场景

- 全新功能开发
- 大规模重构
- 架构升级

## 标准步骤

1. 明确用户观察到的现象
2. 重现问题
3. 区分事实、猜测、未知项
4. 定位最小修复点
5. 修复后验证主流程、空状态、错误状态

## 何时读取附加文档

- 环境问题：读 `../50-验证与交付/上线前环境一致性检查.md`
- 边界混乱：读 `../20-界面规则/功能切分与边界控制.md`

## 最低验证

- 问题可重现或可明确描述
- 修复后至少验证主路径一次
- 不把未验证说成已修复
