# external-api-debug

## 适用场景

- 第三方 API 调不通
- key 可用但请求失败
- 本地浏览器可访问但服务端不可访问
- 结果未入库
- 页面只显示泛化失败

## 不适用场景

- 纯 UI 问题
- 与外部服务完全无关的本地逻辑 bug

## 标准步骤

1. 验证 key / endpoint / method
2. 验证 DNS、443 连通性
3. 验证运行进程是否真正走代理
4. 验证返回状态码和响应体
5. 验证数据写入和页面展示

## 排查顺序

1. 配置问题
2. 网络问题
3. 代理问题
4. 外部 API 权限问题
5. 代码映射问题

## 最低验证

- 至少确认失败卡在哪一层
- 给用户返回人话错误
- 日志里保留技术细节
