# delivery-readiness

## 适用场景

- 用户说准备上线
- 准备交付 / 提测
- 本地已完成，需要判断是否足够稳

## 不适用场景

- 还处于需求不清阶段
- 功能主体还没实现完成

## 标准步骤

1. 构建验证
2. 类型验证
3. 主流程验证
4. 空状态验证
5. 错误状态验证
6. 外部 API 验证
7. 环境一致性检查

## 何时读取附加文档

- 上线差异：读 `../50-验证与交付/上线前环境一致性检查.md`
- 最小检查项：读 `../50-验证与交付/验证清单.md`

## 最低交付标准

- build 通过
- 主路径可跑
- 风险已说明
- 未验证项已明确标注
