# Skills 设计规则

## 总原则

skill 应该短、可维护、按需读。

不要把 skill 写成百科全书。

## 推荐结构

1. `SKILL.md`
   - 只放触发条件
   - 只放最小流程
   - 只放必要说明

2. `references/`
   - 放细节说明
   - 放例子
   - 放专项规范

3. `scripts/`
   - 放重复执行且需要稳定性的脚本

## 语言建议

1. 规则正文优先中文
2. 技术标识、命令、字段名保留英文

## 省 token 规则

1. 常驻内容越短越好
2. 细节放 references
3. 不重复写同一规则
4. 示例最少化
5. 优先清单，不写长篇散文
