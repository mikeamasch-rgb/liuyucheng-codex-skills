# 数据库与迁移规则

## Schema 规则

改数据库前必须明确：

1. 表名
2. 字段
3. 可空性
4. 唯一约束
5. 索引
6. 关系
7. 默认值

## 数据表原则

1. 主结果数据与原始数据分开
2. 用户可见表和内部调试表分开
3. 不用 JSON 代替应当结构化的字段

## 迁移原则

1. schema 改动必须同步到数据库
2. 改完 schema 后要验证 client / ORM 类型是否同步
3. 不假设运行态会自动吃到最新 schema

## 数据展示原则

数据库里有完整字段时：

- 结果区优先展示完整字段
- 不要长期用回退字段代替真实字段
