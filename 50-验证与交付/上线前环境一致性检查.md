# 上线前环境一致性检查

这份文档专门解决：

- 本地正常
- 一上线就坏

## 核心原则

本地可用 != 可上线

上线问题通常来自：

1. 环境变量不一致
2. 网络环境不一致
3. 本地代理依赖
4. 数据库差异
5. 构建和运行方式差异

## 上线前必须检查

### 1. 环境变量

逐项确认：

- 本地 `.env`
- 测试环境 env
- 生产 env

特别检查：

- API keys
- DB URL
- 代理配置
- 第三方 endpoint

### 2. 数据库

确认：

- schema 已同步
- migration 已执行
- 线上数据表存在
- 必需索引存在

### 3. 网络与代理

确认：

- 本地是否依赖代理
- 线上是否也需要访问外部服务
- 线上出站网络是否允许访问目标域名

### 4. 构建方式

确认：

- 本地 `dev` 与生产 `build/start` 是否存在差异
- 是否只在 dev 环境下可用
- 是否依赖本地路径、桌面文件、系统端口

## 最小 smoke test

上线前至少测：

1. 首页能打开
2. 主提交流程可触发
3. 外部 API 路径可通
4. 数据能写库
5. 结果能展示
6. 错误能人话提示

## 常见误区

1. 浏览器能访问 Google，不代表 Node 进程能访问
2. 本地全局代理开着，不代表服务端进程继承到了
3. 本地有数据库，不代表线上 schema 已同步
4. 页面 build 通过，不代表外部 API 真可用
