# 换电脑迁移说明

这份文档解决：

- 换电脑后这套规则库怎么带过去
- 如何保证项目继续按同样方式工作

## 一、最重要的东西是什么

你真正要迁移的不是单个项目，而是这三层：

1. 桌面规则库
2. Codex 本地自定义 skills
3. 项目自己的短版 `AGENTS.md`

## 二、最小迁移清单

### 1. 桌面规则库

复制整个目录：

- `C:\\Users\\admin\\Desktop\\项目规则库`

### 2. 个人 skills

如果你把自己的通用 skill 放在：

- `C:\\Users\\admin\\.codex\\skills`

则需要一起迁移。

### 3. 当前项目

复制项目目录本身，包括：

- 代码
- `AGENTS.md`
- `README`
- `docs`

### 4. 环境配置

单独检查：

- `.env`
- 本地代理工具
- Node / npm
- Docker / 数据库

不要默认新电脑会和旧电脑一致。

---

## 三、换电脑后的接入步骤

### 第一步

把桌面规则库复制到新电脑固定位置。

建议继续放：

- `Desktop/项目规则库`

### 第二步

把自定义 skills 放回：

- `%USERPROFILE%\\.codex\\skills`

### 第三步

打开项目，检查项目 `AGENTS.md` 里的路由路径是否仍有效。

例如：

- `../项目规则库/`

如果桌面位置变了，就要同步改路径。

### 第四步

验证运行环境：

1. `npm install`
2. `build`
3. 数据库连接
4. 外部 API key
5. 代理

---

## 四、换电脑后最容易出问题的点

1. 桌面路径变了，`AGENTS.md` 引用失效
2. 技能目录没迁移
3. 本地代理没装
4. Docker / PostgreSQL 没装
5. `.env` 没同步
6. Node 版本不同

---

## 五、推荐做法

如果以后这套规则库长期使用，最好：

1. 把桌面规则库放入一个可同步目录
2. 自定义 skills 同样做备份
3. 给规则库加一个版本说明或更新日志

这样换电脑时不需要重新从零搭。
