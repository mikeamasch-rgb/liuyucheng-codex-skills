# 工程任务分流

做项目时，先判断任务属于哪一类，再决定读什么。

## 任务分流表

### UI / 页面结构

读：

- `20-界面规则/界面与产品页规则.md`

### API / 后端流程

读：

- `30-后端与接口/API与后端规则.md`

### 数据库 / schema / migration

读：

- `40-数据库/数据库与迁移规则.md`

### 依赖选型

读：

- `60-依赖与选型/依赖选型规则.md`

### Skill 设计与拆分

读：

- `70-skills设计/skills设计规则.md`
- `70-skills设计/工程类skills模板.md`

### 交付前检查

读：

- `50-验证与交付/验证清单.md`

## 先问自己三个问题

1. 这是主流程问题还是实现细节问题
2. 这是业务规则问题还是工程规则问题
3. 这是一次性任务还是可复用工作流

根据答案决定：

- 写 prompt
- 读规则文档
- 还是沉淀成 skill
