# 界面与产品页规则

## 先判断用户要完成什么

做 UI 前先回答：

1. 用户的主任务是什么
2. 用户最先要看到什么
3. 主操作是什么
4. 次级操作是什么
5. 哪些信息是结果，哪些只是内部实现

## 不把内部实现概念暴露给用户

如果某个概念只是技术实现层使用：

- task
- sync job
- raw payload
- execution record

默认不应该成为主页面核心模块，除非用户明确需要。

## 结果页规则

如果页面核心是搜索和结果：

1. 搜索条件区在上
2. 结果区在下
3. 结果字段按业务信息排列
4. 不让内部状态或后台概念抢主视图

## 结果字段优先级

优先展示：

1. 名称
2. 完整地址
3. 联系方式
4. 网址

次级信息：

- 分类
- 评分
- 评论数
- place id

## 视觉原则

1. 低噪音
2. 明确对齐
3. 少装饰
4. 不默认大卡片
5. 优先内容可读性
